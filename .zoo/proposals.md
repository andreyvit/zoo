# Zoo Proposals

- Incoming proposals live under `.proposals/`.
- New proposal files use `.proposals/YYYYMMDD-title.md`, with a lower-hyphen title slug.
- Archived proposals move to `.proposals/archived/`.
- New proposals start with `status: "proposed"` frontmatter. Valid statuses are `draft`, `proposed`, `accepted`, `done`, and `rejected`.
- Proposals require human approval before implementation unless the user explicitly says the proposal is already accepted.
